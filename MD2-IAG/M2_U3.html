Módulo 02 – Aula 3

Aplicações práticas de RAG e Avaliação

Como transformar RAG em um sistema real, avaliar qualidade e evitar erros em produção.

O que muda quando vamos para a prática?

Entender RAG é diferente de colocar RAG em produção.

  • Dados reais são bagunçados
  • Respostas precisam ser confiáveis
  • Custo e latência importam

RAG não é só perguntar e responder

Um sistema RAG completo envolve:

  • Ingestão de dados
  • Processamento e divisão
  • Busca semântica
  • Geração controlada

Pipeline prática de um RAG

  1. Coletar documentos
  2. Dividir em chunks
  3. Gerar embeddings
  4. Armazenar vetores
  5. Consultar e responder

Ingestão de dados

Sistemas RAG trabalham com dados reais:

  • PDFs
  • Sites
  • Planilhas
  • Bases internas

Por que dividir os documentos?

Textos grandes demais:

  • Dificultam a busca
  • Geram respostas genéricas
  • Aumentam custo

Chunking (divisão do texto)

Chunking é dividir documentos em partes menores e significativas.

Cada chunk vira uma unidade de busca.

Indexação vetorial

Cada chunk é transformado em um embedding.

Esses vetores são armazenados em um banco vetorial.

Consulta semântica

A pergunta do usuário também vira um embedding.

O sistema recupera os chunks mais relevantes.

Geração da resposta

A IA recebe:

  • Pergunta do usuário
  • Contexto recuperado

E gera uma resposta contextual.

Exemplo prático

Pergunta: "Quais são os benefícios da empresa?"

O sistema busca no manual interno antes de responder.

Atividade 1

Simular um pipeline RAG completo.

Cada grupo define:

  • Fonte de dados
  • Chunks
  • Pergunta
  • Resposta esperada

Por que avaliar um RAG?

Um RAG pode parecer correto e ainda assim estar errado.

Avaliar evita decisões ruins.

Métricas de avaliação

  • Recall
  • Precisão
  • Latência
  • Custo

Recall

Mede se o sistema encontrou a informação correta.

Precisão

Mede se os resultados recuperados realmente são relevantes.

Latência e custo

Um bom RAG precisa:

  • Responder rápido
  • Custar pouco
  • Escalar bem

Atividade 2

Avaliação manual de um chatbot RAG.

Julgue:

  • Contexto recuperado
  • Qualidade da resposta
  • Possíveis falhas

Ferramentas para RAG

  • LangChain
  • LlamaIndex
  • Streamlit
  • FastAPI

Encerramento

RAG transforma modelos de linguagem em sistemas confiáveis.

Sem avaliação, não existe RAG seguro.
