Boas Práticas de Engenharia de Prompt

IA Generativa aplicada à Programação

Duração: 4h

Objetivo da Unidade

  • Aprender a estruturar prompts eficientes
  • Evitar erros comuns
  • Refinar respostas da IA
  • Usar IA de forma profissional

Boas Práticas Fundamentais

  • Ser claro e direto
  • Definir objetivo
  • Dar contexto
  • Definir formato da resposta
  • Iterar até melhorar

Importância do Contexto

A IA não sabe quem você é, nem o seu nível.
Se você não disser, ela assume.

Exemplo com Contexto

"Sou iniciante em programação. Explique o que é uma API REST usando exemplos simples e linguagem acessível."

Definir Formato da Resposta

  • Lista
  • Passo a passo
  • Código comentado
  • Tabela

Exemplo com Formato

"Explique promises em JavaScript em:
  • Definição
  • Exemplo de código
  • Erro comum
"

Atribuir um Papel à IA

Dizer para a IA quem ela é muda a resposta.

Exemplo de Papel

"Você é um desenvolvedor sênior. Revise este código e sugira melhorias seguindo boas práticas."

Erros Comuns

  • Prompts vagos
  • Não revisar a resposta
  • Copiar código sem entender
  • Confiar cegamente na IA

Exemplo de Prompt Ruim

"Faça um sistema"

❌ Sem contexto

❌ Sem tecnologia

❌ Sem objetivo

Iterar é Obrigatório

O primeiro prompt raramente é o melhor.

Exemplo de Iteração

1️⃣ "Crie uma API"
2️⃣ "Crie uma API REST em Node.js"
3️⃣ "Crie uma API REST em Node.js com Express, usando MVC e comentários no código"

Uso Responsável

  • Validar informações
  • Testar códigos
  • Evitar plágio
  • Manter ética profissional

Atividade 1 – Diagnóstico

Analise este prompt e diga o que falta:

"Explique banco de dados"

Atividade 2 – Refinamento

Comece com um prompt simples e refine até obter uma resposta ideal.

Atividade 3 – Prompt Profissional

Crie um prompt para:
  • Gerar código
  • Revisar código
  • Explicar um erro

Reflexão Final

A IA erra rápido.
O profissional corrige melhor.

Encerramento

  • Prompt é habilidade
  • Iteração é prática
  • Responsabilidade é humana
